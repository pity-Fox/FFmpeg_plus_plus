# FFmpeg++ 节点编辑器使用指南

## 概述

节点编辑器是 FFmpeg++ 的高级视频处理模式。通过可视化的蓝图式画布，你可以构建复杂的视频处理流程——拖拽节点、连线、配置参数，无需手写 ffmpeg 命令。

> 在 **设置 → 编辑模式** 中选择「节点编辑器」启用此功能。选择「传统模式」则使用简单的傻瓜式配置。

---

## 核心概念

### 节点（Node）

每个节点代表一个处理操作。可用节点类型：

| 节点 | 图标 | 说明 |
|------|------|------|
| **源文件** | 🎬 | 处理链的起点，代表输入视频文件。一个编辑器中可创建多个源文件节点，**每个源文件节点代表一个独立任务** |
| **音视频处理** | 🎛️ | 视频编码（H.264/H.265/AV1/VP9）、分辨率缩放、帧率调整、音频编码 |
| **字幕烧录** | 📝 | 将外挂字幕或内嵌字幕轨道硬编码到视频中，支持字体/颜色/描边样式 |
| **片段截取** | ✂️ | 截取视频的指定时间范围（如前 10 分钟） |
| **帧提取** | 📷 | 从视频中提取静态帧（单帧/范围分帧/全部分帧） |
| **输出** | 💾 | 处理链的终点，配置输出格式/文件名/保存目录 |

### 连线（Connection）

节点之间通过连线传递数据流：

- 每个节点左侧有**输入端口**（蓝色圆点），右侧有**输出端口**（主题色圆点）
- 从一个节点的端口区域拖拽到另一个节点即可连线
- 源文件节点没有输入端口，输出节点没有输出端口

### 执行链

一个完整的执行链必须是：**源文件 → 处理节点（一个或多个）→ 输出**

---

## 操作方式

### 画布操作
- **平移画布**：在空白区域按住左键拖拽
- **缩放画布**：滚轮缩放（0.3x ~ 2.0x）
- **添加节点**：在空白处**右键**，弹出菜单选择节点类型
- **删除节点**：在节点上**右键** → 删除节点
- **删除连线**：在连线附近**右键** → 删除连线

### 节点操作
- **移动节点**：拖拽节点的**中间区域**
- **连线**：拖拽节点的**左侧或右侧端口区域**到目标节点
- **编辑参数**：点击节点，右侧面板显示对应的参数编辑器
- 源文件节点点击后右侧显示视频预览图和文件信息

### 保存
- 点击右上角「保存」按钮
- 保存前会**自动验证**逻辑正确性，有错误会弹窗提示

## 逻辑规则

### 必须遵守的规则

1. **完整链条**：每条链必须从源文件开始，到输出结束
2. **无环路**：不允许循环连线（A→B→C→A）
3. **每个节点必须连接**：不能有悬空的节点（无输入或无输出）

### 节点兼容性

| 规则 | 说明 |
|------|------|
| 音视频处理 + 字幕烧录 | ✅ 可以同时使用，会合并为一条 ffmpeg 命令 |
| 音视频处理 + 片段截取 | ❌ 不能在同一层级，必须串联（先处理再截取，或先截取再处理） |
| 音视频处理 + 帧提取 | ❌ 不能在同一层级 |
| 帧提取 → 任何视频节点 | ❌ 帧提取输出为图片，后续不能连接视频处理节点 |
| 片段截取 → 片段截取 | ⚠️ 后续截取的时间范围不能超过前一次截取的结果长度 |

### 合并优化

当同一层级同时存在「音视频处理」和「字幕烧录」节点时，系统会自动将两者合并为**单条 ffmpeg 命令**执行，避免不必要的中间文件和二次编码。

## 执行流程

### 单链执行
```
源文件 → 音视频处理 → 输出
```
生成 1 条 ffmpeg 命令，直接输出最终文件。

### 多步串联
```
源文件 → 音视频处理 + 字幕烧录 → 片段截取 → 输出
```
- 步骤 1：合并执行音视频处理 + 字幕烧录 → 临时文件
- 步骤 2：对临时文件执行片段截取 → 最终输出
- 步骤 3：自动清理临时文件

### 多任务独立
```
源文件A → 压缩 → 输出A
源文件B → 字幕烧录 → 截取 → 输出B
```
两个源文件节点 = 两个独立任务，互不干扰，各自进入处理队列。

---

## 帧提取模式

帧提取支持三种模式：

| 模式 | 说明 |
|------|------|
| **单帧** | 提取指定时间点的一帧图片 |
| **范围** | 提取指定时间段内的所有帧（可设置提取帧率），输出到文件夹 |
| **全部** | 提取整个视频的所有帧（可设置提取帧率），输出到文件夹 |

> 提取多帧时，结果自动保存到以视频名命名的文件夹中。

---

## 调试功能

在 **设置 → 调试模式** 开启后，节点编辑器左下角会用半透明文字实时显示：

- 解析出的任务数量
- 每个任务的执行步骤
- 哪些操作被合并为单条命令
- 中间文件传递关系

用于排查逻辑配置问题。

## 设计理念

### 为什么用节点编辑器？

传统的视频处理工具通常是线性流程——一个个选项从上到下填写。这在简单场景下够用，但面对复杂需求时（如「先压缩+烧字幕，再截取片段」「同一个视频输出多个版本」）就力不从心了。

节点编辑器的核心思想是**将处理流程可视化为有向无环图（DAG）**：

- **节点** = 操作（编码、字幕、截取...）
- **连线** = 数据流向（上一步的输出 → 下一步的输入）
- **多源文件** = 多个独立任务

这让你能直观地构建任意复杂的处理逻辑，而系统自动处理中间文件管理、命令合并优化、执行调度。

### 智能合并

系统不会机械地为每个节点都生成一条 ffmpeg 命令。当检测到「音视频处理」和「字幕烧录」在同一层级时，会自动合并为一条命令（如 `ffmpeg -i input -vf subtitles=... -c:v libx264 ... output`），避免不必要的中间文件和画质损失。

### 验证优先

保存时系统会做全面的逻辑检查——缺少连线、循环引用、类型冲突、时间范围越界等问题都会在保存前被拦截，而不是等到执行时才报错。
