# FFmpeg++ 节点编辑器使用指南

## 概述

节点编辑器是 FFmpeg++ 的高级视频处理模式。通过可视化的蓝图式画布，你可以构建复杂的视频/图片/音频处理流程——拖拽节点、连线、配置参数，无需手写 ffmpeg 命令。

> 在 **设置 → 编辑模式** 中选择「节点编辑器」启用此功能。选择「传统模式」则使用简单的傻瓜式配置。

---

## 核心概念

### 节点（Node）

每个节点代表一个处理操作。系统提供 **25+ 节点类型**，按功能分为以下几组：

#### 🔌 基础节点

| 节点 | 图标 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| **源文件** | 🎬 | — | 视频/音频/图片 | 处理链起点，代表输入文件。多个源文件节点 = 多个独立任务 |
| **输出** | 💾 | 视频/音频/图片 | — | 处理链终点，配置输出格式/文件名/保存目录 |

#### 🎞 视频处理节点

| 节点 | 图标 | 输入 → 输出 | 说明 |
|------|------|-------------|------|
| **音视频处理** | 🎛️ | 视频 → 视频 | 编码（H.264/H.265/AV1/VP9/SVT-AV1 等 17+ 编码器）、分辨率、帧率、音频编码、GPU 加速 |
| **字幕烧录** | 📝 | 视频 → 视频 | 烧录外挂 SRT/ASS/SSA 字幕，支持字体/颜色/描边样式，系统字体选择器（含预览）|
| **片段截取** | ✂️ | 视频 → 视频 | 截取指定时间范围，支持级联时长约束 |
| **帧提取** | 📷 | 视频 → 图片 | 单帧/范围分帧/全部分帧，输出为图片文件 |
| **变速** | ⏩ | 视频 → 视频 | 视频变速处理 |
| **视频裁剪** | ✂️ | 视频 → 视频 | 交互式选区工具，支持多选区、拖拽调整大小、保留/移除模式，保留音轨 |
| **提取音频** | 🎵 | 视频 → 音频 | 从视频中提取音频轨道，支持 copy 模式和转码模式，自动检测编解码器兼容性，带音频预览播放 |

#### 🎵 音频处理节点

| 节点 | 输入 → 输出 | 说明 |
|------|-------------|------|
| **音频转换** | 音频 → 音频 | 音频格式/编码转换 |
| **音质调整** | 音频 → 音频 | 调整音频质量参数 |
| **调整速度** | 音频 → 音频 | 音频变速（不改变音调）|
| **调整音量** | 音频 → 音频 | 音量增益/衰减 |
| **压缩动态范围** | 音频 → 音频 | 音频动态压缩器 |
| **元信息编辑** | 音频 → 音频 | 编辑标题/艺术家/专辑等元数据 |

#### 🖼 图片处理节点

| 节点 | 输入 → 输出 | 说明 |
|------|-------------|------|
| **图片转换** | 图片 → 图片 | 格式转换（PNG/JPG/WebP/BMP 等）|
| **图片裁剪** | 图片 → 图片 | 按坐标裁剪区域 |
| **图片旋转** | 图片 → 图片 | 旋转/翻转 |
| **图片缩放** | 图片 → 图片 | 分辨率缩放 |
| **亮度调节** | 图片 → 图片 | 调节亮度/对比度 |
| **添加噪点** | 图片 → 图片 | 添加噪点效果 |
| **图片锐化** | 图片 → 图片 | 锐化处理 |
| **图片降噪** | 图片 → 图片 | 降噪处理 |
| **通道提取** | 图片 → 图片 | 提取 R/G/B/Alpha 通道 |

#### 🔗 组合节点

| 节点 | 输入 → 输出 | 说明 |
|------|-------------|------|
| **合并媒体** | 视频/音频 → 视频 | 多文件顺序合并 |
| **图片合成视频** | 图片 → 视频 | 图片序列合成为视频 |

### 连线（Connection）

节点之间通过连线传递数据流：

- 每个节点左侧有**输入端口**（蓝色圆点），右侧有**输出端口**（主题色圆点）
- 从一个节点的端口区域拖拽到另一个节点即可连线
- 源文件节点没有输入端口，输出节点没有输出端口
- **类型匹配**：输出类型必须与下游节点的输入类型匹配（视频→视频、音频→音频、图片→图片）

### 执行链

一个完整的执行链必须是：**源文件 → 处理节点（一个或多个）→ 输出**

## 操作方式

### 画布操作
- **平移画布**：在空白区域按住左键拖拽
- **缩放画布**：滚轮缩放（0.3x ~ 2.0x）
- **添加节点**：在空白处**右键**，弹出菜单选择节点类型
- **删除节点**：在节点上**右键** → 删除节点
- **删除连线**：在连线附近**右键** → 删除连线

### 节点操作
- **移动节点**：拖拽节点的**中间区域**
- **连线**：拖拽节点的**左侧或右侧端口区域**到目标节点
- **编辑参数**：点击节点，右侧面板显示对应的参数编辑器
- 源文件节点点击后右侧显示视频预览图和文件信息

### 保存
- 点击右上角「保存」按钮
- 保存前会**自动验证**逻辑正确性，有错误会弹窗提示

## 逻辑规则

### 必须遵守的规则

1. **完整链条**：每条链必须从源文件开始，到输出结束
2. **无环路**：不允许循环连线（A→B→C→A）
3. **每个节点必须连接**：不能有悬空的节点（无输入或无输出）
4. **类型匹配**：连线两端的媒体类型必须兼容

### 节点兼容性

| 规则 | 说明 |
|------|------|
| 音视频处理 + 字幕烧录 | ✅ 可以同时使用，会合并为一条 ffmpeg 命令 |
| 音视频处理 + 片段截取 | ❌ 不能在同一层级，必须串联 |
| 音视频处理 + 帧提取 | ❌ 不能在同一层级 |
| 帧提取 → 视频节点 | ❌ 帧提取输出为图片，不能接视频处理节点 |
| 帧提取 → 图片节点 | ✅ 帧提取输出图片，可接图片处理节点 |
| 提取音频 → 音频节点 | ✅ 提取音频输出音频，可接音频处理节点 |
| 图片节点 → 图片合成视频 | ✅ 图片处理后可合成为视频 |
| 片段截取 → 片段截取 | ⚠️ 后续截取的时间范围不能超过前一次截取的结果长度 |
| 视频裁剪 / 提取音频 / 变速 | ⚠️ 必须独占执行，不能与其他操作并行 |

### 媒体类型流转

```
视频源 → [视频处理节点] → 视频输出
       → [帧提取] → 图片 → [图片处理节点] → 图片输出
       → [提取音频] → 音频 → [音频处理节点] → 音频输出
图片源 → [图片处理节点] → 图片输出
       → [图片合成视频] → 视频输出
音频源 → [音频处理节点] → 音频输出
```

### 合并优化

当同一层级同时存在「音视频处理」和「字幕烧录」节点时，系统会自动将两者合并为**单条 ffmpeg 命令**执行，避免不必要的中间文件和二次编码。

## 执行流程

### 单链执行
```
源文件 → 音视频处理 → 输出
```
生成 1 条 ffmpeg 命令，直接输出最终文件。

### 多步串联
```
源文件 → 音视频处理 + 字幕烧录 → 片段截取 → 输出
```
- 步骤 1：合并执行音视频处理 + 字幕烧录 → 临时文件
- 步骤 2：对临时文件执行片段截取 → 最终输出
- 步骤 3：自动清理临时文件

### 跨类型链
```
源文件(视频) → 提取音频 → 音频转换 → 输出
源文件(视频) → 帧提取 → 图片锐化 → 图片缩放 → 输出
源文件(图片) → 图片裁剪 → 图片合成视频 → 输出
```

### 多任务独立
```
源文件A → 压缩 → 输出A
源文件B → 字幕烧录 → 截取 → 输出B
```
两个源文件节点 = 两个独立任务，互不干扰，各自进入处理队列。

## 特殊节点说明

### 帧提取模式

| 模式 | 说明 |
|------|------|
| **单帧** | 提取指定时间点的一帧图片 |
| **范围** | 提取指定时间段内的所有帧（可设置提取帧率），输出到文件夹 |
| **全部** | 提取整个视频的所有帧（可设置提取帧率），输出到文件夹 |

> 提取多帧时，结果自动保存到以视频名命名的文件夹中。

### 提取音频

- 支持 **copy 模式**（直接复制，无损）和**转码模式**（转换编码格式）
- 自动检测源视频的音频编解码器，判断与目标容器的兼容性
- copy 模式不兼容时自动切换为合适的目标格式，并显示警告
- 内置音频预览播放器，可试听选定的时间段
- 时间范围选择器，支持截取部分音频

### 视频裁剪

- 提取指定帧号的画面作为参考图
- 打开全屏选区工具，在参考图上绘制裁剪区域
- 支持多选区、8 个调整手柄（四角 + 四边中点）、拖拽移动、撤销
- **保留模式**：保留选区内的画面（多选区取并集）
- **移除模式**：移除选区内的画面（保留选区外的部分）
- 裁剪时保留原始音轨（`-c:a copy`）

## 调试功能

在 **设置 → 调试模式** 开启后，节点编辑器左下角会用半透明文字实时显示：

- 解析出的任务数量
- 每个任务的执行步骤
- 哪些操作被合并为单条命令
- 中间文件传递关系

用于排查逻辑配置问题。

## 设计理念

### 为什么用节点编辑器？

传统的视频处理工具通常是线性流程——一个个选项从上到下填写。这在简单场景下够用，但面对复杂需求时（如「先压缩+烧字幕，再截取片段」「同一个视频输出多个版本」「视频提取帧→图片处理→合成新视频」）就力不从心了。

节点编辑器的核心思想是**将处理流程可视化为有向无环图（DAG）**：

- **节点** = 操作（编码、字幕、截取、裁剪、音频处理、图片处理...）
- **连线** = 数据流向（上一步的输出 → 下一步的输入）
- **类型系统** = 视频/音频/图片三种媒体类型的自动匹配与转换
- **多源文件** = 多个独立任务

这让你能直观地构建任意复杂的处理逻辑，而系统自动处理中间文件管理、命令合并优化、执行调度。

### 智能合并

系统不会机械地为每个节点都生成一条 ffmpeg 命令。当检测到「音视频处理」和「字幕烧录」在同一层级时，会自动合并为一条命令（如 `ffmpeg -i input -vf subtitles=... -c:v libx264 ... output`），避免不必要的中间文件和画质损失。

### 验证优先

保存时系统会做全面的逻辑检查——缺少连线、循环引用、类型冲突、时间范围越界等问题都会在保存前被拦截，而不是等到执行时才报错。
